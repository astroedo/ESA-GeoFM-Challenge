# ESA GeoFM Challenge — Embed2Heights
> Multi-Modal Foundation Model Embedding Fusion for Height Estimation and Land Cover Segmentation

This repository contains the end-to-end Machine Learning pipeline developed for the ESA Phi-Lab GeoFM Challenge. The primary objective is to simultaneously predict pixel-level land cover segmentation (buildings, vegetation, water) and Normalized Digital Surface Model (nDSM) height fields from a diverse set of pre-computed embeddings generated by state-of-the-art Geospatial Foundation Models (GFMs).

**Final Validation/Leaderboard Score:** 0.4130 (Achieved via FusionNetV4 architecture in `notebook/ESA_v5_final.ipynb`).

---

## 1. Problem Statement and Evaluation Metrics

Given multi-modal embedding tensors representing a 256×256 Earth observation patch (at a native resolution of 10 meters per pixel), the objective is to map these hidden representations into a 4-channel spatial output target:
* **Channel 0:** Building area fraction (%)
* **Channel 1:** Tree/Vegetation area fraction (%)
* **Channel 2:** Water body area fraction (%)
* **Channel 3:** nDSM height estimation (meters)

The global performance score is a mathematically rigid, weighted linear combination of Mean Intersection over Union (mIoU) and Root Mean Squared Error (RMSE):

$$\text{Score} = 0.25 \cdot \text{mIoU}_{\text{buildings}} + 0.15 \cdot \text{mIoU}_{\text{trees}} + 0.15 \cdot \text{mIoU}_{\text{water}} + 0.25 \cdot \text{RMSE}_{\text{height-buildings}} + 0.20 \cdot \text{RMSE}_{\text{height-veg}}$$

---

## 2. Data Streams and Foundation Model Representations

The design leverages a heterogeneous fusion strategy separating dense spatial arrays from sparse tokenized descriptors:

### Spatial Stream (High-Resolution 256×256 Tensors)
* **AlphaEarth (64 channels):** Captures high-frequency geometric boundaries, serving as the architectural backbone for fine-grained structural delimitation.
* **Tessera (128 channels):** Imparts rich spectral and textural contextualization across dense spatial grids.

### Token Stream (Low-Resolution 16×16 Vision Transformer Tokens)
* **TerraMind S2 (768 channels):** ViT-driven representations providing long-range dependency modeling and macro-semantic scene understanding.
* **THOR S1/S2 (768 channels):** Co-aligned radar and optical features essential for capturing vertical profiles (altimetry) but inherently prone to wide, unstable dynamic ranges.

## 3. Architectural Evolution and Iterative Benchmarking

The development lifecycle progressed through four distinct milestones to empirically validate each architectural module:

1.  **AlphaEarth Baseline (`notebook/train_alphaearth_only.ipynb`):** A lightweight convolutional configuration utilizing only dense high-resolution spatial inputs. While boundary definitions for large buildings were crisp, height estimation suffered heavily from a lack of macro-contextual awareness, resulting in an insufficient global score.
2.  **Dual-Branch Convolutional Fusion (`notebook/train_dual_branch.ipynb`):** An early fusion implementation where the 16×16 TerraMind tokens were compressed via 1×1 convolutions, upsampled using bilinear interpolation to 256×256, and concatenated with the spatial stream. This configuration yielded a validation mIoU of 0.575 and a partial Leaderboard score of 0.4112.
3.  **Cross-Attention Fusion (`notebook/train_attention_fusion.ipynb`):** Replaced static bilinear upsampling with a cross-attention mechanism, allowing dense spatial queries to actively sample the 256 macro-tokens from TerraMind. A custom Gradient Loss term was introduced on the height regression head to heavily penalize blurry structural transitions.
4.  **Production Pipeline v5 — FusionNetV4 (`notebook/ESA_v5_final.ipynb`):** The final production network integrating Squeeze-and-Excitation (SE) blocks within the spatial encoder, an Atrous Spatial Pyramid Pooling (ASPP) bottleneck to capture multi-scale receptive fields (ranging from 8m to 144m on-ground), and decoupled regression heads for urban and vegetative altimetry.

## 4. Hardware Constraints, Memory Optimization, and Numerical Stability

Deploying deep architectures on modern accelerators requires balancing modeling complexity against strict physical hardware constraints. The technical breakthrough achieved in version v5 centers entirely around optimizing GPU throughput and enforcing absolute numerical stability.

### Mixed Precision Debugging and Overflow Remediation
During mixed-precision training executions (`torch.amp.autocast`), earlier iterations frequently suffered from catastrophic gradient explosion (`Loss = NaN`). Forensic analysis isolated the issue to the pre-computed GFM embeddings: certain unnormalized dimensions with extremely low variance exploded under FP16 dynamic scaling limits ($\pm65504$). 

To rectify this, a precise clamping threshold (`FEAT_CLAMP = 15.0`) was implemented globally. Restricting the feature dynamic range to $\pm15$ during both dataset loading and inference phases eradicated dynamic scaling overflows while preserving 100% of the underlying semantic signal.

### GPU Memory Management and I/O Bottleneck Resolution
Training was constrained to a single accelerator runtime (16GB VRAM environment). Storing the comprehensive multi-modal embedding dataset directly on disk caused standard PyTorch DataLoader workers to experience severe I/O thrashing, lowering GPU utilization down to 23%. 

To maximize throughput, the pipeline was redesigned around a dual-tier caching strategy:
* The tokenized representations (TerraMind/THOR) and targets were pre-loaded and locked directly in host RAM as FP16 arrays.
* Heavy spatial tensors (AlphaEarth/Tessera) were streamed from an optimized, local NVMe staging directory.
* This approach reduced data fetch times by a factor of 8, allowing the batch size to scale efficiently to 32 while driving steady GPU compute utilization to over 91%.

### Generalization and Structural Regularization
To counter aggressive overfitting on the validation set, data augmentation was shifted directly into the latent space. Embedding-space Augmentation was deployed by injecting controlled Gaussian noise ($\sigma = 0.03$) alongside a 5% Channel Dropout applied selectively across the pre-trained feature channels. Class-imbalance vulnerabilities were addressed using a `WeightedRandomSampler` to structurally scale up the presentation frequency of scarce, low-density land cover classes.
